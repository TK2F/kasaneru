# OBS 解像度・Browser Source の合わせ方（YouTube / Twitch 想定）

Kasaneru の **ドックとプレビュー**は常に **1920×1080 の論理座標**で動きます。`overlay.html` は **Browser Source に設定した幅・高さ**を実ビューポートとして読み、**均等スケール（contain）＋中央寄せ**で 1920×1080 のステージを収めます。  
YouTube / Twitch でよく使われる **16:9・720p / 1080p** は、そのまま対応できます。

---

## 推奨パターン（どちらか一方で OK）

### A. Browser Source を 1920×1080 にする（最も分かりやすい）

| 項目 | 値 |
|------|-----|
| Browser Source の幅 | **1920** |
| Browser Source の高さ | **1080** |
| シーン上 | キャンバス（ベース）に **フィット**または **均等スケール** で配置 |

- **キャンバス 1920×1080・出力 1920×1080** のとき: そのまま 1:1。
- **キャンバスが 2560×1440 など大きい**とき: ソースを **縮小**して配置すれば、OBS が最終的に 1080p 出力へダウンスケールしても **一様スケール**のまま。

### B. Browser Source を 1280×720 にする（軽めのマシン向け）

| 項目 | 値 |
|------|-----|
| Browser Source の幅 | **1280** |
| Browser Source の高さ | **720** |

内部では **`min(1280/1920, 720/1080)` で均等スケール**され、**見え方は 1080p 設定と同じ比率**になります（ピクセル密度だけ変化）。

---

## プラットフォーム別の「よくある出力」（参考）

| 用途 | 典型的な出力解像度 | 本オーバーレイとの関係 |
|------|-------------------|------------------------|
| YouTube 1080p | 1920×1080 | **A** または **B** でそのまま |
| YouTube 720p | 1280×720 | **B** が自然。**A** のソースをシーンで縮小でも可 |
| Twitch 1080p | 1920×1080 | 同上 |
| Twitch 720p | 1280×720 | 同上 |

エンコードビットレートや FPS は本ドキュメントの対象外です。

---

## 注意（レイアウトが崩れやすい操作）

1. **シーンアイテムで横だけ・縦だけストレッチ**すると、オーバーレイが非一様変形し、ドックのガイドや感覚と一致しません。**クロップ＋均等スケール**を推奨します。
2. **「ローカルファイル」Browser Source** と **ドックの file URL** が **別オリジン**になると BroadcastChannel が繋がらないので、README のとおり **URL 欄に file:///… を入れ、ローカルファイルはオフ**にしてください。
3. **非 16:9 の Browser Source**（例: 余計な余白付き）では、ステージの上下または左右に **透明の余白**が付きます（意図した contain 動作です）。

---

## 検証の目安

- 設定した解像度で **LIVE / TitleCard / 画像**を出し、四隅の余白と角のバランスを確認。
- **720p** と **1080p** の両方で 1 回ずつ、`docs/QA-CHECKLIST.md` の該当項目を回すと安心です。
