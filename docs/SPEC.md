# Kasaneru 仕様書（Software Requirements / Spec）v5.3

> **別名**: 要件定義の参照用として本書を利用（従来の「REQ」と同格）。

## 1. 目的・スコープ

- OBS Studio 上で、チャプター / TitleCard / 画像の **16:9 配信向けオーバーレイ** を、**追加インストールなし**（HTML + CSS + BroadcastChannel）で操作・表示する。
- 主に **YouTube / Twitch** で一般的な **720p / 1080p・16:9** 出力を想定する。**非 16:9 唯一パス**（例: ウルトラワイド単体キャンバス）は「表示は可能だが余白が出る」程度のサポートとする。

## 2. 座標系・解像度（重要）

| 項目 | 仕様 |
|------|------|
| **論理設計サイズ** | **1920 × 1080 px**（オフセット・ガイド・画像スケールの基準） |
| **Browser Source** | ユーザーが任意の **幅 × 高さ** で設定可能（推奨は下記 `BROADCAST-SETUP.md`） |
| **ビューポート適合** | `overlay.html` は `window.innerWidth/Height` に対し **均等スケール（contain）** で `#lt-stage`（1920×1080）を **中央配置**。アスペクトが 16:9 でない場合は **透明の余白**（レターボックス） |
| **キャンバス / 出力** | OBS が最終的に **スケーリング・リサイズ** しても、ソース内では **一様スケール** 前提のため、**シーン上でオーバーレイを非一様に引き伸ばさないこと**（横だけ伸ばす等）は推奨しない（ドックと見え方が一致しなくなる） |

## 3. 機能要件（概要）

- **dock.html**: リスト編集、グループ、設定（テーマ、アクセント、レイヤー、オフセット、ガイド、**CSV（チャプター / タイトルカード / 画像・種別選択と置換インポート）**、Undo、編集ロック、i18n 等）。画面上部は **言語切替のみ（右寄せ）**。
- **overlay.html**: `sync` / `hide` / TitleCard / 画像メッセージに応じた表示・アニメーション。
- **永続化**: `localStorage` キー `lt_ch_v5`。
- **制約**: ES5、CDN 禁止（`CLAUDE.md` 参照）。

## 4. 非機能要件

- OBS 28+ 系の Browser Source（CEF）で動作すること。
- テーマ CSS は GPU フレンドリーな `transform` / `opacity` を主とする（`backdrop-filter` は使用しない）。

## 5. 懸念と明確な非サポート

| 懸念 | 対策 / 方針 |
|------|-------------|
| Browser Source 解像度が 1920×1080 でない | **自動スケール**で吸収（実装済み） |
| キャンバス 1440p・出力 1080p など | ブラウザソースをシーン上で **一様拡大縮小** する通常運用で整合 |
| HDR / 色空間 | OBS 任せ。本仕様では扱わない |
| 4K 出力のみ・UI が小さすぎる | Browser Source を **3840×2160** にするか、シーンで拡大。**内部基準は引き続き 1920×1080 論理** |
| 縦動画 (9:16) | **対象外**。別レイアウトが必要 |
